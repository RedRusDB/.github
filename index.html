RedRusDb

A distributed in-memory k-v storage engine in Rust!

About the Project

An attempt to write a distributed in-memory k-v storage engine in Rust to learn about database internals.

  • Build a database from scratch
  • Learn database internals, starting with Redis
  • Learn about advanced data structures, algorithms, and event loops
  • Collaborate with other engineers and contribute back to Open Source

While our B+ tree now supports concurrent operations, it's still a single-threaded database system, as our frontend (network/cli layer) doesn't support handling concurrent requests yet.

This is by no means an idiomatic Rust implementation as I'm learning Rust along the way.

Project Status

The project is in a slow phase because I have a job in parallel and am learning things on the way. Hence, the progress will be slower. Hopefully, I can regain my momentum after a couple of weeks.

End Goal

The main focus is to write a distributed in-memory k-v storage engine from scratch. This project will include its own:

  • [ ] B+ Tree data structure
  • [ ] Buffer pool
  • [ ] LRU replacement policy
  • [ ] Transaction manager
  • [ ] Lock manager
  • [ ] Vector support
  • [ ] Persistence storage
