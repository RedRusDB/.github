RedRusDb

A distributed in-memory k-v storage engine in Rust!

About the Project

An ambitious journey to craft a distributed in-memory key-value storage engine in Rust, driven by the desire to unravel the intricacies of database internals.

  • Construct a database from the ground up
  • Explore database internals, starting with Redis
  • Master advanced data structures, algorithms, and event loops
  • Collaborate with fellow engineers and contribute to the world of Open Source

While our B+ tree now supports concurrent operations, it's currently a single-threaded database system. Concurrent request handling awaits implementation in our frontend (network/cli layer).

This implementation might not be idiomatic Rust, as I'm learning Rust along the way.

Project Status

The project is progressing at a deliberate pace due to parallel work commitments. Learning is happening in tandem with real-world application. Expect a gradual increase in momentum over the coming weeks.

End Goal

The ultimate aim is to construct a distributed in-memory key-value storage engine from scratch. This project will boast its own:

  • [ ] B+ Tree data structure
  • [ ] Buffer pool
  • [ ] LRU replacement policy
  • [ ] Transaction manager
  • [ ] Lock manager
  • [ ] Vector support
  • [ ] Persistence storage
